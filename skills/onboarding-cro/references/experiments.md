# Onboarding Experiment Ideas （オンボーディング実験のアイデア）

Comprehensive list of A/B tests and experiments for user onboarding and activation. （ユーザーのオンボーディングとアクティベーションのための A/B テストと実験の包括的なリスト。）

## Flow Simplification Experiments （流れの単純化実験）

### Reduce Friction （摩擦を減らす）

| Test（テスト） | Hypothesis（仮説） |
|------|------------|
| Email verification timing（メール認証のタイミング） | During vs. after onboarding（オンボーディング中とオンボーディング後） |
| Empty states vs. dummy data（空の状態とダミーデータの比較） | Pre-populated examples（事前に入力された例） |
| Pre-filled templates（事前に入力されたテンプレート） | Accelerate setup with templates（テンプレートを使用してセットアップを加速する） |
| OAuth options（OAuth オプション） | Faster account linking（より迅速なアカウントリンク） |
| Required step count（必要歩数） | Fewer required steps（必要な手順が少なくなる） |
| Optional vs. required fields（オプションのフィールドと必須フィールド） | Minimize requirements（要件を最小限に抑える） |
| Skip options（スキップオプション） | Allow bypassing non-critical steps（重要ではない手順のバイパスを許可する） |

### Step Sequencing （ステップシーケンス）

| Test（テスト） | Hypothesis（仮説） |
|------|------------|
| Step ordering（ステップの順序付け） | Test different sequences（さまざまなシーケンスをテストする） |
| Value-first ordering（価値優先の順序付け） | Highest-value features first（最も価値の高い機能を最初に） |
| Friction placement（摩擦の配置） | Move hard steps later（難しいステップを後で移動する） |
| Required vs. optional balance（必須とオプションのバランス） | Ratio of required steps（必要なステップの割合） |
| Single vs. branching paths（単一パスと分岐パス） | One path vs. personalized（単一パス vs パーソナライズド） |
| Quick start vs. full setup（クイックスタートと完全セットアップ） | Minimal path to value（価値への最小限のパス） |

### Progress & Motivation （進歩とモチベーション）

| Test（テスト） | Hypothesis（仮説） |
|------|------------|
| Progress bars（プログレスバー） | Show completion percentage（完了率を表示） |
| Checklist length（チェックリストの長さ） | 3-5 items vs. 5-7 items（3 ～ 5 個のアイテムと 5 ～ 7 個のアイテム） |
| Gamification（ゲーミフィケーション） | Badges, rewards, achievements（バッジ、報酬、実績） |
| Completion messaging（完了メッセージ） | "X% complete" visibility（「X% 完了」の可視性） |
| Starting point（出発点） | Begin at 20% vs. 0%（20% 対 0% から開始） |
| Celebration moments（お祝いの瞬間） | Acknowledge completions（完了を確認する） |

## Metrics to Track （追跡する指標）

For all experiments, measure: （すべての実験について、以下を測定します。）

| Metric（メトリック） | Description（説明） |
|--------|-------------|
| Activation rate（活性化率） | % reaching activation event（% アクティベーション イベントに到達） |
| Time to activation（アクティベーションまでの時間） | Hours/days to first value（最初の値までの時間/日数） |
| Step completion rate（ステップ完了率） | % completing each step（各ステップを完了した割合） |
| Drop-off points（降車地点） | Where users abandon（ユーザーが放棄する場所） |
| Return rate（返品率） | Users who come back（戻ってくるユーザー） |
| Day 1/7/30 retention（1/7/30日目の保持） | Engagement over time（長期にわたるエンゲージメント） |
| Feature adoption（機能の採用） | Which features get used（どの機能が使用されるか） |
| Support requests（サポートリクエスト） | Volume during onboarding（オンボーディング中の音量） |
