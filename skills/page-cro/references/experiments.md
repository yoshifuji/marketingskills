# Page CRO Experiment Ideas （ページ CRO 実験のアイデア）

Comprehensive list of A/B tests and experiments organized by page type. （ページの種類ごとに整理された A/B テストと実験の包括的なリスト。）

## Homepage Experiments （ホームページの実験）

### Hero Section （ヒーローセクション）

| Test（テスト） | Hypothesis（仮説） |
|------|------------|
| Headline variations（見出しのバリエーション） | Specific vs. abstract messaging（具体的なメッセージングと抽象的なメッセージング） |
| Subheadline clarity（サブヘッドラインの明確さ） | Add/refine to support headline（サポート見出しを追加/修正する） |
| CTA above fold（スクロールせずに見える位置にある CTA） | Include or exclude prominent CTA（目立つCTAを含めるか除外する） |
| Hero visual format（ヒーローのビジュアルフォーマット） | Screenshot vs. GIF vs. illustration vs. video（スクリーンショット vs. GIF vs. イラスト vs. ビデオ） |
| CTA button color（CTAボタンの色） | Test contrast and visibility（コントラストと視認性をテストする） |
| CTA button text（CTAボタンのテキスト） | "Start Free Trial" vs. "Get Started" vs. "See Demo"（「無料トライアルを開始する」 vs. 「始めてみる」 vs. 「デモを見る」） |
| Interactive demo（インタラクティブなデモ） | Engage visitors immediately with product（訪問者に製品を即座にアピールする） |

### Trust & Social Proof （信頼と社会的証明）

| Test（テスト） | Hypothesis（仮説） |
|------|------------|
| Logo placement（ロゴの配置） | Hero section vs. below fold（ヒーローセクションとスクロールせずに見える部分） |
| Case study in hero（ヒーローのケーススタディ） | Show results immediately（結果をすぐに表示） |
| Trust badges（トラストバッジ） | Add security, compliance, awards（セキュリティ、コンプライアンス、賞を追加） |
| Social proof in headline（見出しの社会的証明） | "Join 10,000+ teams" messaging（「10,000 チーム以上に参加」メッセージ） |
| Testimonial placement（お客様の声の掲載） | Above fold vs. dedicated section（スクロールせずに見える部分と専用セクション） |
| Video testimonials（ビデオによる紹介） | More engaging than text quotes（テキストの引用よりも魅力的） |

### Features & Content （機能とコンテンツ）

| Test（テスト） | Hypothesis（仮説） |
|------|------------|
| Feature presentation（機能のプレゼンテーション） | Icons + descriptions vs. detailed sections（アイコン + 説明と詳細セクション） |
| Section ordering（セクションの順序付け） | Move high-value features up（価値の高い機能を上に移動） |
| Secondary CTAs（セカンダリCTA） | Add/remove throughout page（ページ全体の追加/削除） |
| Benefit vs. feature focus（利点と機能の焦点） | Lead with outcomes（成果をあげてリードする） |
| Comparison section（比較セクション） | Show vs. competitors or status quo（競合他社または現状との比較を示す） |

### Navigation & UX （ナビゲーションとUX）

| Test（テスト） | Hypothesis（仮説） |
|------|------------|
| Sticky navigation（スティッキーナビゲーション） | Persistent nav with CTA（CTA 付きの永続的なナビゲーション） |
| Nav menu order（ナビメニューの順序） | High-priority items at edges（エッジにある優先度の高いアイテム） |
| Nav CTA button（ナビゲーションCTAボタン） | Add prominent button in nav（ナビゲーションに目立つボタンを追加する） |
| Support widget（サポートウィジェット） | Live chat vs. AI chatbot（ライブチャットとAIチャットボット） |
| Footer optimization（フッターの最適化） | Clearer secondary conversions（より明確な二次変換） |
| Exit intent popup（インテントポップアップを終了する） | Capture abandoning visitors（放棄された訪問者を捕捉する） |

## Pricing Page Experiments （価格ページの実験）

### Price Presentation （価格提示）

| Test（テスト） | Hypothesis（仮説） |
|------|------------|
| Annual vs. monthly display（年間と月間の表示） | Highlight savings or simplify（節約を強調するか簡素化する） |
| Price points（価格帯） | $99 vs. $100 vs. $97 psychology（99 ドル対 100 ドル対 97 ドルの心理学） |
| "Most Popular" badge（「最も人気のある」バッジ） | Highlight target plan（目標計画を強調表示する） |
| Number of tiers（階層数） | 3 vs. 4 vs. 2 visible options（3 対 4 対 2 の表示オプション） |
| Price anchoring（価格アンカーリング） | Order plans to anchor expectations（期待を裏付ける注文計画） |
| Custom enterprise tier（カスタムエンタープライズ層） | Show vs. "Contact Sales"（表示 vs. 「営業担当者へのお問い合わせ」） |

### Pricing UX （価格設定のUX）

| Test（テスト） | Hypothesis（仮説） |
|------|------------|
| Pricing calculator（価格計算ツール） | For usage-based pricing clarity（使用量ベースの価格設定を明確にするため） |
| Guided pricing flow（ガイド付きの価格設定フロー） | Multistep wizard vs. comparison table（複数ステップのウィザードと比較表） |
| Feature comparison format（特徴比較フォーマット） | Table vs. expandable sections（テーブルと展開可能なセクションの比較） |
| Monthly/annual toggle（月次/年次切り替え） | With savings highlighted（節約が強調表示された状態） |
| Plan recommendation quiz（企画おすすめクイズ） | Help visitors choose（訪問者の選択を支援する） |
| Checkout flow length（チェックアウトフローの長さ） | Steps required after plan selection（プラン選択後に必要な手順） |

### Objection Handling （異議申し立てへの対応）

| Test（テスト） | Hypothesis（仮説） |
|------|------------|
| FAQ section（よくある質問セクション） | Address pricing objections（価格設定に関する異議申し立てに対処する） |
| ROI calculator（ROI 計算ツール） | Demonstrate value vs. cost（価値とコストを比較する） |
| Money-back guarantee（返金保証） | Prominent placement（目立つ配置） |
| Per-user breakdowns（ユーザーごとの内訳） | Clarity for team plans（チーム計画を明確にする） |
| Feature inclusion clarity（特徴の包含の明確さ） | What's in each tier（各層の内容） |
| Competitor comparison（競合他社の比較） | Side-by-side value comparison（値を並べて比較） |

### Trust Signals （トラストシグナル）

| Test（テスト） | Hypothesis（仮説） |
|------|------------|
| Value testimonials（価値ある証言） | Quotes about ROI specifically（特にROIに関する引用） |
| Customer logos（顧客のロゴ） | Near pricing section（価格セクション付近） |
| Review scores（レビュースコア） | G2/Capterra ratings（G2/カプテラの評価） |
| Case study snippet（ケーススタディのスニペット） | Specific pricing/value results（具体的な価格設定/価値の結果） |

## Demo Request Page Experiments （デモリクエストページの実験）

### Form Optimization （フォームの最適化）

| Test（テスト） | Hypothesis（仮説） |
|------|------------|
| Field count（フィールド数） | Fewer fields, higher completion（フィールドが少なく、完成度が高い） |
| Multi-step vs. single（マルチステップとシングルステップの比較） | Progress bar encouragement（プログレスバーの促進） |
| Form placement（フォームの配置） | Above fold vs. after content（アバブ コンテンツとアフター コンテンツ） |
| Phone field（電話フィールド） | Include vs. exclude（含めるか除外するか） |
| Field enrichment（フィールドの強化） | Hide fields you can auto-fill（自動入力できるフィールドを非表示にする） |
| Form labels（フォームラベル） | Inside field vs. above（インサイドフィールド vs. 上記） |

### Page Content （ページコンテンツ）

| Test（テスト） | Hypothesis（仮説） |
|------|------------|
| Benefits above form（フォーム上の特典） | Reinforce value before ask（尋ねる前に価値を強化する） |
| Demo preview（デモのプレビュー） | Video/GIF showing demo experience（デモ体験を示すビデオ/GIF） |
| "What You'll Learn"（「何を学ぶか」） | Set expectations clearly（期待を明確に設定する） |
| Testimonials near form（フォームに近いお客様の声） | Reduce friction at decision point（意思決定ポイントでの摩擦を軽減） |
| FAQ below form（フォームの下のFAQ） | Address common objections（よくある反対意見に対処する） |
| Video vs. text（動画とテキストの比較） | Format for explaining value（値を説明する形式） |

### CTA & Routing （CTAとルーティング）

| Test（テスト） | Hypothesis（仮説） |
|------|------------|
| CTA text（CTAテキスト） | "Book Your Demo" vs. "Schedule 15-Min Call"（「デモを予約する」 vs. 「15 分間の通話をスケジュールする」） |
| On-demand option（オンデマンドオプション） | Instant demo alongside live option（インスタントデモとライブオプション） |
| Personalized messaging（パーソナライズされたメッセージング） | Based on visitor data/source（訪問者のデータ/ソースに基づく） |
| Navigation removal（ナビゲーションの削除） | Reduce page distractions（気を散らすページを減らす） |
| Calendar integration（カレンダーの統合） | Inline booking vs. external link（インライン予約と外部リンクの比較） |
| Qualification routing（資格のルーティング） | Self-serve for some, sales for others（セルフサービスもあれば、販売もある） |

## Landing Page Experiments （ランディング ページの実験）

### Message Match （メッセージマッチ）

| Test（テスト） | Hypothesis（仮説） |
|------|------------|
| Headline matching（見出しの一致） | Match ad copy exactly（広告コピーを正確に一致させる） |
| Visual matching（ビジュアルマッチング） | Match ad creative（広告クリエイティブの一致） |
| Offer alignment（オファーの調整） | Same offer as ad promised（広告で約束したものと同じオファー） |
| Audience-specific pages（対象者別のページ） | Different pages per segment（セグメントごとに異なるページ） |

### Conversion Focus （コンバージョンフォーカス）

| Test（テスト） | Hypothesis（仮説） |
|------|------------|
| Navigation removal（ナビゲーションの削除） | Single-focus page（単焦点ページ） |
| CTA repetition（CTAの繰り返し） | Multiple CTAs throughout（全体に複数の CTA） |
| Form vs. button（フォームとボタン） | Direct capture vs. click-through（直接キャプチャとクリックスルーの比較） |
| Urgency/scarcity（緊急性/希少性） | If genuine, test messaging（本物の場合、メッセージングをテストする） |
| Social proof density（社会的証明の密度） | Amount and placement（金額と配置） |
| Video inclusion（ビデオの組み込み） | Explain offer with video（動画で特典を解説） |

### Page Length （ページの長さ）

| Test（テスト） | Hypothesis（仮説） |
|------|------------|
| Short vs. long（短いものと長いもの） | Quick conversion vs. complete argument（迅速な変換と完全な引数） |
| Above-fold only（スクロールせずに見える範囲のみ） | Minimal scroll required（最小限のスクロールが必要） |
| Section ordering（セクションの順序付け） | Most important content first（最も重要なコンテンツを最初に） |
| Footer removal（フッターの削除） | Eliminate navigation（ナビゲーションを排除する） |
